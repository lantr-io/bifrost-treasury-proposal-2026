# scalus-treasury

Operational setup for the Cardano treasury withdrawal funding the
**Scalus Application Platform and Node Diversity** initiative.

> **Status:** review draft. The proposal at [`docs/proposal.md`](docs/proposal.md)
> is still being iterated on; the on-chain submission has not happened yet.

## What this repo is

A small TypeScript codebase that parameterizes and drives the audited
[SundaeSwap `treasury-funds`](https://github.com/SundaeSwap-finance/treasury-contracts)
Aiken contracts to:

1. mint a one-shot registry NFT pinning our treasury+vendor script hashes;
2. register both scripts as DReps and delegate them to `AlwaysAbstain`
   (constitutional requirement — withdrawn ADA cannot vote);
3. emit a CIP-108 governance-action JSON to be submitted via `cardano-cli`;
4. (after ratification) withdraw funds from the treasury reward account into
   a treasury-script UTxO;
5. (per project) fund a vendor contract with a milestone schedule;
6. (per milestone) claim matured payouts.

This repo does **not** contain contract code. The contracts are upstream and
audited; we consume them via the
[`@sundaeswap/treasury-funds`](https://www.npmjs.com/package/@sundaeswap/treasury-funds)
npm package. A pinned reference checkout of the contract source lives at
[`contracts/treasury-contracts`](contracts/treasury-contracts) (submodule,
audited commit).

## Architecture

```
                docs/proposal.md
                       │
         build-anchor → pin → blake2b-256
                       │
         ┌─────────────┼─────────────┐
         ▼             ▼             ▼
   on-chain        gov-action    cardano-cli
   anchor URL        JSON        submission
                       │
                       ▼
            CIP-1694 treasury withdrawal
                       │
                       ▼
       ┌────── treasury reward account ──────┐
       │                                      │
       ▼                                      ▼
  treasury.ak  ── Fund ──▶  vendor.ak  ── Withdraw ──▶ vendor wallet
  (operator + board)         (per-project schedule)
```

- **Audited contracts** (Aiken): `treasury.ak`, `vendor.ak`, `oneshot.ak` —
  upstream at SundaeSwap, audited by TxPipe and MLabs (PDFs under
  [`contracts/treasury-contracts/audits/`](contracts/treasury-contracts/audits/)).
- **Bindings** (TS): generated by SundaeSwap from the Aiken `plutus.json`
  blueprint, published as `@sundaeswap/treasury-funds`.
- **Operator scripts** (this repo): hand-written TypeScript that drive the
  bindings.

## Permission topology

| Action                | Required signers                              |
|-----------------------|-----------------------------------------------|
| `treasury.reorganize` | operator                                      |
| `treasury.disburse`   | operator + 1-of-3 oversight board             |
| `treasury.sweep`      | operator + 1-of-3 oversight board             |
| `treasury.fund`       | operator + 2-of-3 board (+ vendor multisig)   |
| `vendor.pause`        | 1-of-3 board                                  |
| `vendor.resume`       | 2-of-3 board                                  |
| `vendor.modify`       | operator + 2-of-3 board (+ vendor multisig)   |

Adapted from
[Blink Labs' Dingo deployment](https://github.com/blinklabs-io/treasury-proposal).
Full rationale, time bounds, and pkh slots in
[Plan.md](Plan.md#treasury-withdrawal-script-address-mainnet-topology).

## Quick start

Requires [nix](https://nixos.org/download.html) with flakes enabled. From a
clean clone:

```sh
git submodule update --init
nix develop                  # or `direnv allow` if you use direnv
bun install                  # postinstall patches libsodium
bun test
bun run typecheck
```

The full preprod end-to-end runbook (anchor pinning, gov-action submission)
lives in [Plan.md](Plan.md).

Operator scripts default to `--dry-run` (build the tx, print a summary, do
not submit). Submission requires the explicit `--submit` flag.

## Trust model

- We trust the SundaeSwap `treasury-funds` contracts (TxPipe + MLabs audited).
  We do **not** re-audit, port, or fork them.
- The contract submodule is pinned at the audited commit (`dea9e52`,
  TxPipe-findings merge) so the audit reports apply directly.
- The on-chain CIP-100 anchor hash is `blake2b-256` over the bytes of
  `gov/anchor.preprod.json`, built reproducibly from
  [`docs/proposal.md`](docs/proposal.md) — re-run `bun run build-anchor` to
  verify a published anchor.
- Any change to the proposal body that affects the anchor must be re-pinned
  and the gov-action re-submitted; the deposit covers a single anchor hash.
